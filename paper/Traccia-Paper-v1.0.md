# Traccia: An Execution-Centric Benchmark for AI Agents with Residual Evaluation

**Authors**: Traccia Project  
**Repository**: [github.com/d87skg/traccia](https://github.com/d87skg/traccia)  
**Dataset**: [`vasdvae/traccia-benchmark`](https://huggingface.co/datasets/vasdvae/traccia-benchmark) on HuggingFace  
**Tool**: `pip install traccia-eval`

---

## Abstract

Current AI agent benchmarks evaluate only final outputs—accuracy, win-rate, pass@k—while ignoring execution quality. We propose **Execution-Centric Evaluation**, a paradigm where an agent's score derives from its full execution trajectory, not just its final answer. We introduce **Traccia**, an open-source framework that captures structured execution traces (`.evidence`), decomposes them into a 4‑dimensional residual vector (temporal, semantic, structural, recovery), and scores agents on how they handle failures, not just whether they avoid them. We release a benchmark dataset of 27 execution traces across 6 runtimes, a standardized scoring function, and a public leaderboard. Experiments show that output-only rankings diverge from execution-based rankings, revealing that agents achieving the same answer do so through vastly different—and measurably distinct—execution strategies.

---

## Key Claims

1. **Execution traces are a first-class evaluation primitive.** An agent's behavior during task execution—tool selection, failure handling, recovery strategy—contains evaluable structure independent of the final answer.
2. **Execution quality can be quantified through residual analysis.** A 4‑dimensional residual vector (temporal, semantic, structural, recovery) captures deviation from expected execution patterns and feeds directly into scoring.
3. **Output-only rankings diverge from execution-based rankings.** Identical correctness can mask significant differences in execution quality, invisible to traditional benchmarks.

## Quick Start

```bash
# Install core tools
pip install traccia-guard   # safety guard for agents
pip install traccia-eval    # evaluation & leaderboard

# Generate benchmark data with closed-loop execution
traccia loop run --iterations 10

# Score a dataset and view leaderboard
traccia-eval dataset.jsonl

# Load the published benchmark dataset
python -c "from datasets import load_dataset; print(load_dataset('vasdvae/traccia-benchmark'))"
```

## The .evidence Format

`.evidence` is a structured, chain‑verified execution trace packaged as a ZIP archive. Each package contains:

- `session.json` — task objective, runtime, status
- `events.jsonl` — time‑ordered sequence of canonical events
- `evidence.json` — proofs of execution (tool outputs, logs)
- `attribution.json` — actor responsibility mapping
- `signature.sig` — SHA‑256 hash chain integrity verification

Canonical event types unify outputs from diverse agent runtimes (LangChain, CrewAI, OpenAI SDK, Claude Code, OpenHands) into a single evaluation vocabulary.

## Residual Analysis & Scoring

We extract a 4‑dimensional residual vector for each execution trace:

| Dimension    | What it captures                                      |
|--------------|-------------------------------------------------------|
| Temporal     | Event clustering, duplicate timestamps, batching      |
| Semantic     | Unexpected fields in tool outputs, schema drift       |
| Structural   | Hash chain breaks, orphan events                      |
| Recovery     | Ratio of recovery attempts to errors, retry logic     |

The **Traccia Score** is a weighted sum of residual‑derived metrics:

```
Score = 0.3 × Correctness + 0.2 × Stability + 0.2 × Recoverability + 0.2 × (1 − Entropy) + 0.1 × Efficiency
```

This scoring function is deterministic, human‑label‑free, and reproducible.

## Benchmark Dataset

[`vasdvae/traccia-benchmark`](https://huggingface.co/datasets/vasdvae/traccia-benchmark) contains 27 execution traces across 6 runtimes + closed‑loop synthetic data:

| Runtime      | Paradigm           | Traces |
|--------------|--------------------|--------|
| LangChain    | Tool‑calling       | 4      |
| CrewAI       | Multi‑agent        | 4      |
| AutoGen      | Conversational     | 3      |
| OpenAI SDK   | Function‑calling   | 3      |
| Claude Code  | OS‑hybrid          | 4      |
| OpenHands    | Full OS agent      | 4      |
| STG‑Loop     | Closed‑loop synth  | 5      |

Each entry includes full task definition, execution trajectory, residual vector, and evaluation scores.
